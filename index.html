Rouven Reichert

Mechanical engineer at heart, software engineer with passion

Hello there! My name is Rouven, I have a M.Sc. in mechanical engineering where I specialized in robotics and found my passion for mobile robotics.
Since 2020, I am working as a back end developer at SYNAOS where I can combine my interest for software development with my passion for mobile robots by working on the VDA5050: the communication protocol between mobile robot fleet management systems and the robots themselves.

Throughout my entire time at university, I was in touch with programming - originally starting with MATLAB for data analysis and evaluation and later first steps in C as part of a smaller project.
Working on the first projects and lectures in mobile robotics introduced me to the Robot Operating System (ROS) for which I installed and used my first Linux distro (Ubuntu 14.04), acquainted myself with the command line and developed C++ programs that were rather close to the hardware, including state machines and sensor data processing.
Nowadays, I professionally develop Java applications using the Spring framework 5. Some technologies I use daily are:

  • Java (specifically Java 11)
  • Spring framework 5
  • PostgreSQL 15
  • Apache Kafka
  • MQTT
  • Docker
  • Kubernetes
  • Linux and various command line utilities

Lately, I've been also trying to get a grip on front end technologies to better understand the full picture of software applications since they have become a vital part of everyday life long ago.
This homepage is one of the results of this endeavour to further practice everything I pick up along the way and to document my journey. After seeing seasoned developers work highly effectively and confident on the command line (thanks to my awesome colleagues!), I also found my love for vim - I try to write this entire web page using vim to one day come close to that proficiency.
Since I'm a huge geek when it comes to tools like this and get hooked quite easily, I very recently started getting into configuring vim more to my personal preferences. If you like, have a look at my dotfiles repository to check how my vimrc (and potentially other configs) evolve over time! :-)
